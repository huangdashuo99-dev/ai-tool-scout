# CEO Review - Strategic Challenge

提供 CEO/创始人视角的战略审查。挑战范围，找到10星产品，确保你在构建正确的东西。

## Checklist

1. **Read design doc** - understand what's being proposed
2. **Challenge the problem** - is this the right problem?
3. **Find the 10-star product** - what would a 10/10 version look like?
4. **Scope challenge** - too much? too little? wrong direction?
5. **4 modes** - Expansion, Selective Expansion, Hold Scope, Reduction
6. **Write review** - update design doc with CEO feedback

## The Review Process

### Step 1: Read the Design Doc
- Read `docs/specs/YYYY-MM-DD-<topic>-design.md`
- Understand: problem, constraints, proposed solution, approaches

### Step 2: Challenge the Problem
Ask yourself:
- "Is this the most important problem to solve?"
- "Does this align with our goals?"
- "What's the opportunity cost of building this?"

### Step 3: Find the 10-Star Product
Imagine the best possible version:
- What would make this a 10/10 product?
- What would users love that they haven't asked for?
- What's the "magic moment" we're missing?

### Step 4: Determine Mode

| Mode | When to Use | Action |
|------|-------------|--------|
| **Expansion** | Too narrow, missing key features | Add scope, find the real product |
| **Selective Expansion** | Core is right, missing essentials | Add 2-3 critical features only |
| **Hold Scope** | Current scope is correct | Validate and proceed |
| **Reduction** | Too ambitious, trying to do too much | Cut to MVP, ship fast |

### Step 5: Write Review

Update the design doc with:
- CEO verdict (which mode)
- Strategic reasoning
- Specific changes (what to add/remove)
- Priority ranking of features
- Success metrics

## Key Principles

- **Think like a founder** - what would you invest in?
- **Find the wedge** - smallest thing that creates value
- **Kill features ruthlessly** - less is more
- **Focus on distribution** - how will users find this?
- **Time to value** - how fast until users benefit?

## After Review

- Design doc updated with CEO feedback
- Ready for engineering-review to assess technical feasibility
